Information Assurance & Security

A look at the foundational principles and frameworks that protect information systems.

What is Information Assurance?

**Information Assurance (IA)** is the practice of protecting information and information systems to ensure they maintain **Confidentiality, Integrity, and Availability**. This is achieved by implementing a comprehensive, well-thought-out security program.

Information Security Frameworks

A security framework is a documented blueprint that defines the policies and procedures for managing security controls and reducing vulnerabilities within an enterprise.

Certifications & Architectures

Common Criteria (CC)

An international standard for evaluating the security of IT products. It assigns an **Evaluation Assurance Level (EAL)**, from EAL 1 (functionally tested) to EAL 7 (formally verified).

Enterprise Security Architecture (EISA)

A structured approach for building an organization's security architecture by adopting and implementing relevant frameworks.

Types of Security Policies

Policies can be categorized by their level of restrictiveness.

Promiscuous

No restrictions. All services and traffic are allowed.

Permissive

Some restrictions, but only on known threats and attacks.

Prudent

Blocks all services by default unless explicitly allowed. Common in unclassified government facilities.

Paranoid

Restricts everything, with little to no internet connectivity. Used in highly classified environments.

The implementation of strong policies and a well-defined architecture is key to a robust and mature security posture.
